API 문서
이 화면은 포트폴리오 시연을 위한 공개 데모 화면입니다. 실제 운영 환경에서는 관리자 인증, 권한 제어, 감사 로그, IP 제한, CSRF 방어가 필요한 영역입니다.
추천 시연 순서 정상 결제 완료 → 부분취소 → 승인 결과불명 → RecoveryTask 확인 → 정산 생성

결제 운영에서 확인하는 흐름

결제 완료 후 결제 거래가 저장되고 SALE 결제매출 원장이 생성됩니다. 취소 완료 시에는 기존 SALE을 수정하지 않고 CANCEL 취소매출 음수 거래를 별도로 생성합니다.

PG 응답이 확정되지 않은 거래는 승인 결과불명 또는 취소 결과불명 상태로 관리합니다. 외부전송과 알림톡은 결제 저장과 분리하고, 실패나 망취소 필요 상황은 RecoveryTask로 추적합니다.

시나리오 버튼을 누르면 처리 결과가 이 영역에 표시됩니다.

    수동 결제 승인/취소 테스트

    시나리오 실행은 미리 정의된 운영 케이스를 재현하고, 수동 테스트는 주문번호와 금액을 직접 입력해 결제 승인/취소 흐름을 확인하는 영역입니다.

    대기

    결제 거래 내역

    PG 승인/취소 결과와 결제 거래 상태를 확인합니다.

    결제완료: 승인 성공, 매출 원장 반영 대상 부분취소: 일부 금액 취소 완료 취소완료/실패: 취소 완료 또는 실패 상태 결과불명/망취소: PG 결과 확인 또는 운영 복구 필요 대기: 아직 처리 전 상태

    후속처리와 복구 작업

    승인 결과불명과 취소 결과불명 상태에서는 SALE/CANCEL 원장을 바로 생성하지 않습니다. RecoveryTask로 남긴 뒤 재조회로 결과가 확정된 경우에만 매출 원장과 후속처리 Queue를 생성합니다.

    PG 승인 성공 후 내부 저장 또는 후속처리가 실패한 경우에는 망취소 필요 상태로 RecoveryTask를 생성해 운영자가 추적할 수 있도록 구성했습니다.

    대기: 처리 전 또는 재처리 대기 성공: 후속처리 완료 실패: 전송 또는 발송 실패 복구필요: 운영자 확인 또는 배치 재처리 대상

    외부전송 대기함

    매출 거래 이후 외부 운영 시스템으로 전송해야 할 대상을 관리합니다.

    알림톡 발송 대기함

    결제/취소 이벤트 이후 고객 알림톡 발송 대상을 관리합니다.

    복구 작업

    승인 결과불명, 취소 결과불명, 망취소, 외부전송/알림톡 재처리 대상을 추적하고 운영자가 처리합니다.
