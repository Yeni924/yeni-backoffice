ISMS 심사 대응 및 취약점 조치 운영 서비스 보안 점검 과정에서 취약점 조치 요청이 발생했고, 기존 기능을 유지하면서 보안 기준을 맞춰야 했습니다. 취약 구간을 확인하고 입력값 검증, 권한/접근 흐름, 보안 기준에 맞는 수정 사항을 운영 영향 범위 안에서 반영했습니다. 보안 심사 대응과 운영 안정성을 함께 고려한 유지보수 경험
SQL Injection 위험 구간 개선 동적 SQL 사용 구간에서 입력값이 SQL 문장에 직접 결합될 수 있는 위험이 있었습니다. Dapper 기반 파라미터 바인딩을 적용해 입력값과 SQL 문장을 분리하고, 반복되는 조회/수정 로직의 안전성을 높였습니다. 레거시 SQL을 다루면서 보안성과 유지보수성을 함께 개선한 경험
개인정보 동의 / 고객 확인 흐름 정리 고객 동의 이력, 인증 코드 발송, 주소지 입력, POS API의 동의 고객 리스트 조회가 여러 시스템을 거쳐 처리되어야 했습니다. 고객, 서버, DB, POS API, POS 간 처리 순서를 시퀀스 기준으로 정리하고 연동 담당자와 확인 기준을 맞췄습니다. 운영 프로세스를 문서화하고 시스템 간 데이터 흐름을 정리한 경험
운영 요청 기반 기능 개선 운영자가 사용하는 관리자 화면에서 상품, 카테고리, 프로모션, 상태값 관련 수정 요청이 수시로 발생했습니다. 요청 의도를 확인한 뒤 화면 입력 흐름, 검증 조건, 저장/조회 API, POS/KIOSK 반영 기준을 함께 점검해 반영했습니다. 작은 운영 요청도 데이터 정합성과 외부 연동 영향까지 고려해 처리한 경험
