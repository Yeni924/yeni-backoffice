01 완료
프로젝트 기본 구조 및 경력 대시보드 구성

Spring Boot 기반 백오피스 화면 구조, 프로젝트 목록, 상세 모달, DB 명세 화면, Swagger API 문서 연결을 구성했습니다.

02 완료
Mock PG 승인/취소 기본 흐름 구현

실제 PG 운영망과 연결하지 않고, Mock PG Client를 통해 승인/취소 성공 및 실패 흐름을 재현했습니다. 승인 성공 시 결제 거래와 매출 SALE 거래가 생성됩니다.

03 완료
매출 거래 원장 구조 구현

승인 건은 SALE, 취소 건은 CANCEL 음수 거래로 저장합니다. 원매출을 직접 수정하지 않고 거래 이력을 보존하는 방식입니다.

04 진행 중
외부 전송 실패/재처리 흐름 구현

매출 거래 생성 이후 외부 전송 대상을 만들고, Mock 전송 결과에 따라 성공/실패/재전송 상태를 관리하는 기능을 보강하고 있습니다.

05 진행 중
알림톡 Queue / Mock Agent 연결

결제 승인, 취소, 외부 전송 실패 이벤트를 알림톡 Queue로 연결하고, Mock Agent가 Queue를 읽어 발송 성공/실패와 재시도 이력을 남기는 구조를 구현하고 있습니다.

06 진행 예정
매출/정산 통계 및 성능 테스트

SALE/CANCEL 거래 원장을 기준으로 매출 통계를 조회하고, 100만 건 테스트 데이터를 넣어 일반 조회, View, 집계 테이블 방식의 성능 차이를 비교할 예정입니다.

07 진행 예정
정산 배치 및 수수료 정책 확장

PG사/결제수단별 수수료 정책을 관리하고, 미정산 매출을 기준으로 DRAFT 정산 데이터를 생성하는 배치 흐름을 확장할 예정입니다.

08 진행 예정
테스트 코드 및 README 정리

결제 승인/취소, 매출 생성, 외부 전송 실패, 알림톡 발송 실패, 정산 배치 흐름을 테스트 코드와 README로 정리할 예정입니다.
